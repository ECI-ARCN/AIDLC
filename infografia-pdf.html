📋 Anatomía de un Spec (Mega-Prompt)

Un Spec sin los 4 componentes genera código impredecible.

① Contexto
Stack tecnológico, arquitectura existente, equipo y restricciones de negocio.
② Restricciones
Reglas de seguridad, límites de rendimiento, frameworks prohibidos o mandatorios.
③ Criterios
Criterios de Aceptación verificables y medibles (BDD / Gherkin).
④ Casos de Borde
Escenarios de falla: rate limits, timeouts, inputs maliciosos, estados vacíos.

⚡ Vibe Coding vs. Spec-Driven

La diferencia entre un prototipo desechable y código de producción.

❌ Vibe Coding ✅ Spec-Driven
"Hazme un login con validación" Contexto + stack + restricciones + criterios + casos borde
Prototipo con localStorage, sin rate limit Código prod. con httpOnly cookies y manejo de errores
Deuda técnica inmediata Spec versionado = contrato de equipo auditable

🚫 Anti-Patrones Críticos

Errores que degradan la calidad del software generado con IA.

Confirm Trap (Sesgo IA)
Aceptar el primer output sin validar. Riesgo: Código con vulnerabilidades o lógica incorrecta no testeada.
Context Window Overflow
Ignorar el límite del contexto del LLM. Riesgo: Respuestas incoherentes o incompletas en proyectos grandes.
Shadow AI (Uso No Gobernado)
Equipos usando IAs sin políticas. Riesgo: Datos sensibles expuestos, cumplimiento regulatorio violado.
Token Sprawl (Descontrol)
Escalar sin monitorear consumo. Riesgo: Sobrecostos masivos y falta de trazabilidad (FinOps).
Fuga de Contexto (Context Leak)
Pasar secretos o IP al prompt de IAs públicas. Riesgo: Brecha de seguridad y violaciones regulatorias.

Patrones de Arquitectura AIDLC

✓
RAG para Onboarding: Uso de Retrieval-Augmented Generation conectando wikis para que la IA responda dudas de dominio del proyecto sin alucinaciones.
✓
Shadow AI Testing: La IA escribe pruebas unitarias y de mutación en paralelo (Pair Testing), buscando casos límite no contemplados por el desarrollador.
✓
Automated AI Red Teaming: IAs "atacantes" integradas en DevSecOps para auditar y buscar vulnerabilidades lógicas en código generado por asistentes.
